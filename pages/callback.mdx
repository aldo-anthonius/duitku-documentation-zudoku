Duitku will send a response in the form of an HTTP POST return value. The merchant is required to provide an endpoint to receive the callback response from Duitku.

### Configuration & Whitelist

1. Please add the following Duitku outgoing IPs to your server's whitelist requirements:

   **Production:** `182.23.85.8`, `182.23.85.9`, `182.23.85.10`, `182.23.85.13`, `182.23.85.14`, `103.177.101.184`, `103.177.101.185`, `103.177.101.186`, `103.177.101.189`, `103.177.101.190`

2. To configure the callback URL on the Duitku system, please contact us via Live Chat or email at support@duitku.com.

## Parameters

The Callback payload follows the SNAP BI standard format.

- **Method:** `HTTP POST`
- **Content-Type:** `application/json`

| Parameter | Type | Mandatory | Description |
| :--- | :--- | :--- | :--- |
| **OriginalPartnerReferenceNo** | String(64) | Optional | Original transaction identifier on the service consumer system. |
| **OriginalReferenceNo** | String(64) | Optional | Original transaction identifier on the service provider system (Duitku). |
| **OriginalExternalId** | String(19) | Optional | Original Customer Reference Number. |
| **LatestTransactionStatus** | String(2) | Mandatory | Transfer Status Code:<br/>`00` - Success<br/>`01` - Initiated<br/>`02` - Paying<br/>`03` - Pending<br/>`04` - Refunded<br/>`05` - Canceled<br/>`06` - Failed<br/>`07` - Not found |
| **Amount** | Object | Optional | Object value providing amount information. |
| **Amount.Value** | String | Mandatory | Net amount of the transaction. For IDR, the value includes 2 decimal digits (e.g., IDR 10.000 will be formatted as `10000.00`). |
| **Amount.Currency** | String(3) | Mandatory | Currency code as in ISO 4217 (e.g., `IDR`). |
| **BeneficiaryAccountName** | String(100) | Mandatory | The beneficiary account name. |
| **BeneficiaryAccountNo** | String(34) | Mandatory | The beneficiary account number. |
| **BeneficiaryBankCode** | String(11) | Mandatory | The beneficiary bank code. |
| **SourceAccountNo** | String(19) | Mandatory | Your source account number used for the transaction. |
| **TransactionDate** | String(25) | Mandatory | Transaction date in ISO 8601 format (e.g., `YYYY-MM-DDThh:mm:ss+07:00`). |
| **AdditionalInfo** | Object | Optional | Additional information for custom use that is not provided by SNAP. |

<br/>

:::info{title="Callback Notes (Masih belum tau)"}
- **Security:** Ensure you validate the incoming callback using the **SNAP Symmetric Signature** (`X-SIGNATURE`) provided in the request headers to verify it originated from Duitku.
- **Retry Mechanism:** If your server does not respond with an HTTP status `200 OK`, we will resend the notification (maximum 5 times).
- **Pending Status:** For transaction callbacks with status codes like `02` (Paying) or `03` (Pending), please wait or periodically check until H+1 (next working day) to obtain the final transaction status.
- **Manual Verification:** You can actively pull updates on the transaction status by calling the **[Check Status API](/check-status)**.
- Please contact us via Live Chat or email at support@duitku.com if you have further questions.
:::
