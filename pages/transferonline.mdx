The Online Transfer (Disbursement) API allows you to send funds automatically from your Duitku merchant balance to your customers' bank accounts or e-wallets. 

Before performing a transfer, ensure that your merchant balance is sufficient, as our API will validate fund availability before processing. You can use the **Check Balance API** to verify your current balance programmatically.

## Transaction Flow

The disbursement process operates asynchronously and consists of two mandatory steps:
1. **Inquiry:** Validating the destination bank account.
2. **Transfer:** Executing the fund transfer based on the validated inquiry.

---

## Step 1: Validate Account (Inquiry)

Always initiate the disbursement process by performing an Inquiry. This step verifies that the destination bank account or e-wallet is active and valid, preventing failed transfers and unnecessary fees.

**Endpoints:**
- `Sandbox` : POST `https://sandbox.duitku.com/webapi/api/disbursement/inquirysandbox`
- `Production` : POST `https://passport.duitku.com/webapi/api/disbursement/inquiry`

### Inquiry Signature Generation
To authorize the request, you must include a `signature` parameter. Generate this by hashing the following concatenated string using **SHA256**:

> `email` + `timestamp` + `bankCode` + `bankAccount` + `amountTransfer` + `purpose` + `secretKey`

:::tip{title="Where do I get the secretKey?"}
The `secretKey` is equivalent to the **ApiKey** used during the disbursement process. 
You will receive this key directly via email from the Duitku team once you have successfully applied for and activated the disbursement feature for your merchant account.<br /><br />

Don't have a Duitku account yet? **[Register here](https://dashboard.duitku.com/Account/Register)** to get started.
:::

:::info
**Important formatting notes:**
- **E-Wallets:** The `bankAccount` value must use the customer’s registered phone number in a standard format, such as: +62XXXXXXXXX, 62XXXXXXXXX, and 08XXXXXXXXX. 
- We strongly recommend using the `08XXXXXXXXX` format to minimize input errors (over `+62` or `62`).
- **Amount:** The `amountTransfer` must be a whole integer without decimals (e.g., `50000`).
:::

## Step 2: Execute Transfer

Once the Inquiry returns a successful response, you will receive a `disburseId`. You must use this ID to trigger the actual fund transfer.

**Endpoints:**
- `Sandbox` : POST `https://sandbox.duitku.com/webapi/api/disbursement/transfersandbox`
- `Production` : POST `https://passport.duitku.com/webapi/api/disbursement/transfer`

### Transfer Signature Generation
The signature formula for the transfer execution requires more parameters from your inquiry response. Hash the following concatenated string using **SHA256**:

> `email` + `timestamp` + `bankCode` + `bankAccount` + `accountName` + `custRefNumber` + `amountTransfer` + `purpose` + `disburseId` + `secretKey`

:::danger{title="Note"}
If you receive a response code **TO**, **68**, or **-100**, it indicates a critical issue or timeout during the bank processing. 
**DO NOT retry the transfer process programmatically**, as it may result in double deductions. Please contact us via Live Chat or email at support@duitku.com to verify the transaction status.
:::
