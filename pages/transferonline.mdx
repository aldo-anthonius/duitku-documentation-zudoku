This API provides several main functions, including:

- Inquiry Request  
- Transfer Request  
- Transfer Status Check  
- Check Balance  
- Bank List  

To simplify integration, we provide a collection that merchants can use to test API requests and responses through the API Reference tab.

## Online Transfer Transaction Flow

The disbursement process consists of two main steps: inquiry (disbursement inquiry API) and transfer (disbursement transfer API).  
Before performing a transfer, ensure that the merchant balance is sufficient, as our API will validate availability before processing.  
Two additional APIs are the check balance API, which helps verify the current merchant balance, and the inquiry status API to check the current transaction status (e.g., success or failure).

## Transfer Online Inquiry Request

The first step in the disbursement process is initiating an inquiry to the Duitku system.  
You can create a disbursement request by performing an inquiry using this API.

**Method** : `HTTP POST`  
**Type** : `application/json`  

**Development** : `https://sandbox.duitku.com/webapi/api/disbursement/inquirysandbox`  
**Production** : `https://passport.duitku.com/webapi/api/disbursement/inquiry`  

### Request Parameters

| Parameter | Type | Required | Description | Example |
| :--- | :--- | :---: | :--- | :--- |
| **userId** | int | Y | Merchant ID, obtained after the disbursement feature is activated by Duitku. | 3551 |
| **amountTransfer** | int | Y | Amount to be transferred (without decimal format). | 50000 |
| **bankAccount** | string(255) | Y | Bank account number. | For testing, see bankAccount [here](/sandboxtesting) |
| **bankCode** | string(3) | Y | Destination or recipient bank code. | Bank code list can be found [here](#) |
| **email** | string(255) | Y | Merchant email used for Duitku registration. | test@chakratechnology.com |
| **purpose** | string(255) | N | Transaction description or message. | Test Transfer Online Inquiry with Duitku |
| **timestamp** | double | Y | *Unix timestamp* in milliseconds. | 1506486841000 |
| **senderId** | int | N | Merchant customer ID. | 123456789 |
| **senderName** | string(255) | N | Customer name. | |
| **signature** | string(255) | Y | Transaction identification code. Generated by hashing parameters using SHA256. | Formula: SHA256(email + timestamp + bankCode + bankAccount + amountTransfer + purpose + secretKey) |

### Response Parameters

| Parameter | Type | Description | Example |
| :--- | :--- | :--- | :--- |
| **email** | string | Email sent during the inquiry process. | test@chakratechnology.com |
| **bankCode** | string | Bank code sent during the inquiry process. | 014 (example for BCA) |
| **bankAccount** | string | Bank account number sent during the inquiry process. | For testing, see bankAccount [here](/sandboxtesting) |
| **amountTransfer** | int | Amount sent during the inquiry process. | 50000 |
| **accountName** | string | Recipient account holder name. | Test Account |
| **custRefNumber** | string | 9-digit customer reference number printed when the transaction is successful. | 123456789 |
| **disburseId** | string | Duitku disbursement ID. Please store this ID for tracking purposes. | |
| **responseCode** | string | Response code from Duitku. Details can be found [here](/responsecode). | |
| **responseDesc** | string | Description of the response code. | |

:::info
Note:
- If the Inquiry Request returns a response other than success, the merchant may retry the inquiry request.
- For e-wallet transfers, the `bankAccount` value must use the customer’s registered phone number in standard formats such as: +62XXXXXXXXX, 62XXXXXXXXX, or 08XXXXXXXXX.
- We recommend using the 08XXXXXXXXX format to minimize input errors.
:::

## Transfer Online Transfer Request

After receiving a successful response from the inquiry, the next step is to send the required data for the disbursement transfer.

**Method** : `HTTP POST`  
**Type** : `application/json`  

**Development** : `https://sandbox.duitku.com/webapi/api/disbursement/transfersandbox`  
**Production** : `https://passport.duitku.com/webapi/api/disbursement/transfer`  

### Request Parameters

| Parameter | Type | Required | Description | Example |
| :--- | :--- | :---: | :--- | :--- |
| **disburseId** | string (255) | Y | Duitku disbursement ID obtained from the inquiry response. | 12345 |
| **userId** | int | Y | User ID sent during the inquiry. | 3551 |
| **email** | string(255) | Y | Merchant email used for Duitku registration. | test@chakratechnology.com |
| **bankCode** | string(3) | Y | Destination or recipient bank code. | Bank code list can be found [here](#) |
| **bankAccount** | string(255) | Y | Bank account number. | For testing, see bankAccount [here](/sandboxtesting) |
| **amountTransfer** | int | Y | Amount to be transferred (without decimal format). | 50000 |
| **accountName** | string(255) | Y | Recipient account holder name obtained from the inquiry response. | Test Account |
| **custRefNumber** | string(255) | Y | Customer reference number obtained from the inquiry response. | 000000001278 |
| **purpose** | string(255) | Y | Transfer description. | Test Transfer Online Inquiry with Duitku |
| **timestamp** | double | Y | *Unix timestamp* in milliseconds. | 1507024378000 |
| **signature** | string(255) | Y | Transaction identification code generated using SHA256 hashing. | Formula: SHA256(email + timestamp + bankCode + bankAccount + accountName + custRefNumber + amountTransfer + purpose + disburseId + secretKey) |

### Response Parameters

| Parameter | Type | Description | Example |
| :--- | :--- | :--- | :--- |
| **email** | string | Email sent during the inquiry process. | test@chakratechnology.com |
| **bankCode** | string | Bank code sent during the inquiry process. | 014 (example for BCA) |
| **bankAccount** | string | Bank account number sent during the inquiry process. | For testing, see bankAccount [here](/sandboxtesting) |
| **amountTransfer** | int | Amount sent during the inquiry process. | 50000 |
| **accountName** | string | Recipient account holder name obtained from the inquiry response. | Test Account |
| **custRefNumber** | string | 9-digit customer reference number printed when the transaction is successful. | 123456789 |
| **responseCode** | string | Response code from Duitku. Details can be found [here](/responsecode). | |
| **responseDesc** | string | Description of the response code. | |

:::danger{title="Note:"}
- If you receive a response code TO, 68, or -100, it indicates an issue during the transfer process. It is not recommended to retry the process.
- Please contact us via Live Chat or email at support@duitku.com for further transaction status verification.
:::
