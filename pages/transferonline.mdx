The Online Transfer (Disbursement) API allows you to send funds automatically from your Duitku merchant balance to your customers' bank accounts or e-wallets. 

Before performing a transfer, ensure that your merchant balance is sufficient, as our API will validate fund availability before processing. You can use the **Check Balance API** to verify your current balance programmatically.

## Transaction Flow

The disbursement process operates asynchronously and consists of two mandatory steps:
1. **Inquiry:** Validating the destination bank account.
2. **Transfer:** Executing the fund transfer based on the validated inquiry.

![Disbursement Flow](/images/flowtransferonline.webp)

---

## Step 1: Validate Account (Inquiry)

Always initiate the disbursement process by performing an Inquiry. This step verifies that the destination bank account or e-wallet is active and valid, preventing failed transfers and unnecessary fees.

**Endpoints:**
- `Sandbox` : POST `https://sandbox.duitku.com/webapi/api/disbursement/inquirysandbox`
- `Production` : POST `https://passport.duitku.com/webapi/api/disbursement/inquiry`

### Authorization (SNAP Symmetric)
To authorize the request, you must include the standard SNAP BI headers, including the `X-SIGNATURE`. 

Since this is a transactional endpoint, the signature must be generated using the **Symmetric Signature (HMAC_SHA512)** method.

👉 **[Read the full Symmetric Signature guide and formula here ➔](/authentication#symmetric)**

:::info{title="Important formatting notes:"}
- **E-Wallets:** The `BeneficiaryAccountNo` value must use the customer’s registered phone number in a standard format, such as: +62XXXXXXXXX, 62XXXXXXXXX, and 08XXXXXXXXX. 
- We strongly recommend using the `08XXXXXXXXX` format to minimize input errors.
- **Amount:** The `Value` inside the `Amount` object must be a string with 2 decimal digits (e.g., `"50000.00"`).
:::

**[🔗 View full Inquiry Request parameters and code examples in the API Reference ➔](/api-reference/transfer-online#inquiry-request)**

---

## Step 2: Execute Transfer

Once the Inquiry returns a successful response, you must execute the transfer. You must use the exact same `PartnerReferenceNo` that you used in the Inquiry request to trigger the actual fund transfer.

**Endpoints:**
- `Sandbox` : POST `https://sandbox.duitku.com/webapi/api/disbursement/transfersandbox`
- `Production` : POST `https://passport.duitku.com/webapi/api/disbursement/transfer`

### Authorization (SNAP Symmetric)
Similar to the Inquiry process, the Transfer execution also requires standard SNAP BI headers and a signature generated using the **Symmetric Signature (HMAC_SHA512)** method.

👉 **[Read the full Symmetric Signature guide and formula here ➔](/authentication#symmetric)**
